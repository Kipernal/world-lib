template<typename inputIteratorType >
bool worldlib::romUsesSA1 ( inputIteratorType  romStart,
inputIteratorType  romEnd 
)

Tests if the ROM uses the SA-1 chip. Functions use this automatically when determining addresses so you probably won't need to use it manually.

Checks 0x7FD5 and 0x7FD6 as PC addresses for their SA-1 status. Returns true for those addresses. This is the only place where PC addressing is used. For this reason, this library is not compatible with HiROM.

Parameters
romStartAn iterator pointing to the start of the ROM data
romEndAn iterator pointing to the end of the ROM data
Returns
True if the ROM should use SA-1 addressing, false otherwise.
Exceptions
std::runtime_errorThe ROM did not contain this data...but if your ROM doesn't contain this information you don't really have a ROM because something's gone horribly wrong.
